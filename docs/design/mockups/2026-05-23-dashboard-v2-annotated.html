① Header: Lab + Sync + Avatar
Sync badge sostituisce il badge "LIVE". Mostra: verde = aggiornato, arancio = stale >5min, rosso = offline. Si aggiorna ogni 30s. Non ha toggle dark/light — quello è solo in /impostazioni.
② Greeting: Playfair Display
Nome utente in Georgia/Playfair 22px weight 300 (non DM Sans). È l'unico elemento "serif" fuori dai KPI. Data in DM Sans 11px var(--t2).
③ SpotlightCard: SOLO se urgenza
Appare SOLO se esiste segnalazione aperta O lavoro in_ritardo urgente. Se nessuna urgenza → va diretto ai KPI. Non sostituisce la vecchia sezione segnalazioni — la rimpiazza con una card hero.
④ KPI Grid 2×2 — filtri attivi
NON è la vecchia strip scrollabile. Ogni card è un Link Next.js che naviga a lista pre-filtrata. Valore 0 → grigio non cliccabile, nessun chevron. Valore >0 → colore semantico + chevron.
⑤ Colori KPI semantici
Rosso = in_ritardo · Gold = da_fatturare · Grey = materiali/zero · Blue = consegne oggi. Numeri: Georgia 38px weight-300 (Playfair nel CSS finale).
⑥ Urgenze Lab — top 3
Sostituisce la sezione "Pagamenti Scaduti" con totale. Solo le 3 urgenze maggiori. Il totale aggregato non serve qui — è confusione visiva. Cliccabile → /fatture?filter=scadute.
⑦ Materiali: progress bar reale
Bar da scorta_attuale / scorta_minima. Rosso = 0, arancio = sotto soglia. Nessun hardcoded.
⑧ Nav pill: tooltip permanente
Il tooltip "Nuovo lavoro" è sempre visibile (non solo hover). Long press 500ms → editMode drag&drop. Desktop: barra orizzontale in basso.

Dark mode — Regole assolute
Sfondo: #1A1916 (carbonio con tinta calda, NON #1E1E1E grigio neutro)
Testo: #F0EDE8 (avorio, NON bianco puro #FFF o grigio #CCC)
Rosso: #E8001A (+9% luminosità vs #D90012 light per contrasto su scuro)
Separatori in dark mode
var(--border) = rgba(255,255,255,.06) (invertito rispetto al light rgba(0,0,0,.06)). Zero border come sostituto ombre — solo le ombre fanno il lavoro di separazione visiva.
Ombre dark: molto più pesanti
Light: ombre marroni morbide 0.22-0.44 opacity.
Dark: ombre nere 0.52-0.60 opacity con white highlight quasi nullo (0.04-0.05). Questo crea profondità visiva su sfondo scuro.
Sync badge: stale (arancio)
In questo screenshot il badge mostra "5 min fa" con dot arancio (warning). Utente sa che i dati potrebbero essere vecchi. Se offline → rosso + "Offline".

Quando appaiono i tab?
SOLO per utenti con: ruolo=titolare E record in tabella tecnici con attivo=true. Un titolare puro NON vede i tab — vede tutto in un'unica colonna.
Default: sempre Produzione
Al caricamento apre sempre il tab Produzione (lavori operativi). Rimosso il time-based default (mattina→Produzione, pomeriggio→Gestione) — troppa complessità per il gain. localStorage ricorda l'ultima scelta esplicita.
Tab design: --prs background
Background tab container: var(--prs, #D4CFC9) + shadow inset. Tab attivo: var(--elv) + var(--sh-b). Stessa logica del segmented control iOS.
Task list: progress reale
Progress bar = COUNT(fasi dove eseguita_at IS NOT NULL) / COUNT(fasi totali) × 100. Fallback se nessuna fase: ricevuto=10%, in_lavorazione=40%, in_prova=70%, pronto=95%. Non più 0% hardcoded.
