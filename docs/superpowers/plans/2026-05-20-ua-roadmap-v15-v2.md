# UÀ PWA — Piano Maestro V1.5 → V2 → V3

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Portare UÀ dalla parità operativa con DentalMaster (V1.5) al vantaggio competitivo assoluto sul mercato italiano dei laboratori odontotecnici (V2).

**Architecture:** Next.js 16 App Router + Supabase + TailwindCSS v4 + shadcn/ui. Mobile-first PWA con WebAuthn, FAB-driven navigation, design system warm panna v2.2. Tutti i componenti devono funzionare perfettamente su telefono (390px) prima di qualsiasi altra viewport.

**Tech Stack:** Next.js 16, TypeScript, Supabase (PostgreSQL + Auth + Storage + Realtime), TailwindCSS v4, shadcn/ui, Motion 12.x, React PDF, Resend, Stripe, Cloudflare Workers.

**Due principi NON negoziabili:**
1. **Mobile/tablet first**: nato per lo schermo del telefono. Max 3 tap per le operazioni quotidiane. Touch targets ≥ 44px. Nessun hover-only interaction.
2. **Semplicità + Completezza**: l'odontotecnico vede subito quello che gli serve, trova tutto ciò di cui ha bisogno. Progressive disclosure: semplice in superficie, completo in profondità.
