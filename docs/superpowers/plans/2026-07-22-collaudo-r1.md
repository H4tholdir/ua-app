// src/lib/nav/torna-indietro.ts
// Direttiva permanente di Francesco (22/07/2026, CLAUDE.md §9): il back, ovunque nella PWA,
// torna alla pagina PRECEDENTE. Il fallback scatta solo senza storia (deep-link, shortcut PWA,
// notifica push): `history.length` è il segnale migliore disponibile — Next non espone lo stack.
// Firma duck-typed (back/push) e non `useRouter` di proposito: testabile senza Next.
export function tornaIndietro(
  router: { back: () => void; push: (href: string) => void },
  fallback = '/dashboard'
): void {
  if (typeof window !== 'undefined' && window.history.length > 1) {
    router.back()
  } else {
    router.push(fallback)
  }
}
```

### Task 3: Tap Android sulla cassetta (P9) — il click naturale non va ingoiato dopo micro-movimento

**Files:**
- Modify: `src/components/ds/Cassetta.tsx` (`handleClick`, ~riga 204; nuovo ref `ultimoPointer`)
- Test: `tests/unit/Cassetta.test.tsx` (describe nuovo)

**Root cause (ipotesi primaria, da confermare col RED):** su Chrome Android un tap frettoloso porta jitter di 8-15px: oltre la NOSTRA soglia (`SOGLIA_MOVIMENTO_PX = 8` → `spostato = true`, niente `onTap` al pointerup) ma DENTRO lo slop di sistema del browser, che quindi EMETTE il click naturale. `handleClick` oggi lo ingoia (`if (spostato.current) return`) → tap perso. Su iOS il jitter è filtrato prima → funziona. Chiave: **dopo un VERO scroll il browser non emette click sull'elemento** — quindi su touch un click che arriva è per definizione un tap.

**Invarianti da NON rompere (test esistenti):** tap fermo = 1 sola azione (pointerup serve, click sintetico ingoiato) · long-press/drag: click post-sollevamento ingoiato · attivazione AT (click puro) → onTap · tastiera → onTap senza doppioni.

// Cassetta.tsx — nuovo ref accanto agli altri (dopo `tapGestito`):
const ultimoPointer = useRef<string | null>(null)

// in handlePointerDown, prima riga del corpo:
ultimoPointer.current = evento.pointerType

// handleClick — il ramo `spostato` ingoia SOLO i puntatori che emettono click dopo un
// trascinamento (mouse/pen). Su TOUCH il browser non emette click dopo uno scroll: se il
// click arriva, il gesto era un tap dentro lo slop di sistema — è il tap Android che
// il collaudo device (P9) ha trovato perso. `spostato` si azzera in ogni caso.
function handleClick() {
  if (sollevata.current) {
    spostato.current = false
    sollevata.current = false
    return
  }
  if (spostato.current) {
    const eraTouch = ultimoPointer.current === 'touch'
    spostato.current = false
    if (!eraTouch) return
    onTap()
    return
  }
  if (tapGestito.current) {
    tapGestito.current = false
    return
  }
  onTap()
}
```

/** Collaudo R1 (P11c) — la faccia custom deriva luce/ombra dall'hex con un CLAMP di luminanza:
 *  sotto 0.08 (nero, blu notte, ecc.) il derivato SCHIARISCE — `color-mix(…, black)` su una base
 *  già nera è nero-su-nero e la cassetta diventa piatta (collaudo device 22/07). */
export function derivaFacciaCustom(hex: string): string {

  return `linear-gradient(180deg, ${hex}, color-mix(in srgb, ${hex} 72%, black))`
}

### Task 7: La home mostra tutto il contenuto (P3)

**Files:**
- Modify: `src/components/features/home/HomeV3.tsx` (blocco `<style>` ~righe 129-143)
- Documentare: emendamento §3.3 in `docs/design/decisions/2026-07-22-collaudo-r1-emendamenti.md` (nuovo, breve)

**Root cause:** §3.3 prescrive la home NO-SCROLL (`@media max-767: height:100dvh; overflow:hidden`). Sul device reale il contenuto (saluto + striscia + pile + foot col TastoPiu) sfora il viewport e l'`overflow:hidden` TAGLIA — Francesco lo percepisce come «il tasto + copre le pile». Il fix mantiene il no-scroll finché il contenuto ci sta e degrada a scroll SOLO quando non ci sta (reale > ideale); il foot rispetta la safe-area.

- [ ] **Step 1: fix CSS** — nel blocco `<style>` di HomeV3:

```css
/* PRIMA */
@media (max-width: 767px) { .ua-home { height: 100dvh; overflow: hidden; } } /* §3.3 no-scroll */

/* DOPO — Collaudo R1 (P3): il no-scroll resta l'intento (§3.3), ma quando il contenuto
   sfora il viewport la home DEVE poter scorrere invece di tagliare le pile sotto il
   TastoPiù (collaudo device 22/07). min-height + overflow-y:auto = no-scroll quando ci
   sta, scroll naturale quando non ci sta. */
@media (max-width: 767px) { .ua-home { min-height: 100dvh; height: auto; overflow-y: auto; } }
```

E sul `.foot`:

```css
/* PRIMA */
.ua-home .foot { margin-top: 16px; display: flex; flex-direction: column; align-items: center; gap: 8px; }
/* DOPO — il tasto non poggia mai sulla barra gesti (safe-area) */
.ua-home .foot { margin-top: 16px; display: flex; flex-direction: column; align-items: center; gap: 8px;
                 padding-bottom: env(safe-area-inset-bottom); }
```
